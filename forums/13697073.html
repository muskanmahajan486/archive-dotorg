Should be a sensor with 'level' type (I'm assuming you're using KNX with DPT 5.001) that is linked to your dim value read command (KNX group address to DPT 5.001 and 'STATUS' as command).

Another command is needed for setting the dim value, which is a regular command with KNX group address and command value 'SCALE' ('DIM' works too).

Then put them together into a slider with the above sensor and command.

On the UI side, drag a slider on your screen and bind it to the slider construct you created above.

Sorry it's not really documented or fully tested yet.

Note that the deployed version of designer is buggy and duplicate slider/sensor constructs may appear. If that happens, log out and log back in again, that should fix it. If you don't, chances are you end up with a broken XML.

Ok, for 3.007 use one-way buttons as UI widgets (you can't get status from 3.007) and create KNX commands with 'DIM INCREASE' and 'DIM DECREASE' (Designer 2.0 - Create KNX Write Command)

It is hard coded to a maximum dim value of 7 per step at the moment. This may still be less than what you'd like (probably depends on the light but to me it still feels like a small step). One way to increase the step is to create a macro with the KNX dim step command executed twice and then bind the macro to your UI button instead.
