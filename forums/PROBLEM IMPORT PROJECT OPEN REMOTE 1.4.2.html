I tried to track down this problem. It seems to be part of Google's Docs service. Today it seems to work again, maybe they fixed the bug, but to be sure, here is the problem I found:

Google offers a new version of text editor. By default this is active for all newly uploaded documents. If you upload an XML file (as OR KNX does) as a plain text file, Google Docs imports this properly and also lets you edit it. If you try to download it however, you will always get the file as PDF regardless of the format you choose. OR KNX certainly cannot handle PDFs as import data and is kind of upset. You can however disable this new editor in Google Docs' settings (choose settings->editing and disable the checkbox on "Create new documents using the latest version of the document editor").

I will keep an eye on this, maybe it's only a couple of servers in Google's cloud. If the error should return, I will file a bug report for Google.
