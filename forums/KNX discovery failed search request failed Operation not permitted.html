Hmm, permissions issue on Linux?

The discovery is trying to access your network interface and will try to use multicast on the NIC – check if multicast is enabled first and if the Java VM process can access the network interface normally (could try as root if that makes a difference?).

Also, there's a new alternative KNX implementation available here:

https://openremote.svn.sourceforge.net/svnroot/openremote/branches/project/Controller_2_0_0_Alphas-KNXIP/

You can also try if that resolves the issue you're seeing (if you're comfortable building new releases yourself).

Build Instructions (change paths accordingly) : Building OpenRemote 2.0 Developer Releases
