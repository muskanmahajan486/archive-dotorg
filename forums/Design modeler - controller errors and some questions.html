It will require some additional implementation work (at least the more generic case, not just KNX) since driving the UI behavior (which is not specific to any particular protocol) will have to understand some semantic detail of the protocol/commands it is dealing with.

There are no known issue with long command or device names.

However, the tests regarding this are still very poor or missing. So I can't say with certainty that no issues exist (certainly, the database schema imposes some limit on very long names but this should be somewhere in order of 256 characters or longer).

If you find a specific case of long names or using spaces in names that definitely causes a reproducible error, then please let us know and we can try to address it.
