I use VPN. I have an VPN server running on my LAN side – currently I use the one which is installed on my Synology NAS. On Android or iOS I simply use the built in VPN client. After establishing connection on my LAN servers, including the OR controller are visible.
An alternative solution would be to use port forwarding and connecting to your external IP (you can use DynDNS service to tie it to some meaningful name). The disadvantage of this solution is that it is less secure than VPN because all your client server traffic goes unencrypted.

No, I don't mean that you have to use the web console, I just want to progress step by step so I can get a better idea of what might be going wrong.

The iOS console will use HTTP REST calls to the controller, it's the way the communication works.
So I wanted you to use a web browser (Safari on your iPhone) to double check that HTTP calls can reach the controller.

THe Pro version remote access is a service that allows easy access from anywhere without the need to setup a VPN. Unfortunately, it's not ready for prime time yet.
If you have a VPN setup, you don't need that feature. Once a VPN connection is established, the controller should appear "as a local resource" to the console.
