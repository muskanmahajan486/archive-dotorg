Depends on the model of your IP camera and your installation setup.

Today not all streaming options are yet covered.

What is available is a <web> element which can display content from a device or a web server using standard HTTP URL and content types. We're using this for video output with a camera that supports direct web browser access. It is not generic enough to cover all streaming use cases though, and its still in the development branch.

Hi all and sorry for my late reply,

I looked at the AXISM1011 and it supports M-JPEG as well as H264. So for now, M-JPEG should work using the web element.
This element is not yet supported in the modeler, so you need to manually edit the generated XML before installing in the controller.
You can find more discussion on the web element in this thread http://openremote.org/pages/viewpage.action?pageId=15532976

There is no need to link a sensor to the web element unless you want to be able to change the URL (e.g. cycling through different cameras).
