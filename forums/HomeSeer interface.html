Hello OpenRemote

I use HomeSeer and wrote a very simple but very flexible TCP/IP interface to it for use with MediaPortal (a media centre open source project).

It comprises a simple HomeSeer server client and (for MP) a client GUI - but you can drive it using just Telnet! All you have to do is connect to it and then send an xml fragment (details are on the forum I think).

If you want you can use it and adapt it for OpenRemote - it is open source and if you search on MediaPortal's forum you'll find it.

I like the idea of OpenRemote but sorry to say haven't yet found a convincing reason to swap out from using HomeSeer as my base program. Maybe I just haven't looked into it enough, I should spend some time since I do feel that HomeSeer is a bit restrictive. Shame it has other areas that are so flexible (especially voice using internal telephones).
