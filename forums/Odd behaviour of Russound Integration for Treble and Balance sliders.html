Anyone see this behaviour before?

Posted by mredeker at Dec 29, 2014 10:30

Can you check if the following change from me have been merged back? It adds a time-out so that the implementation does not block, should it not receive an acknowledge.

------------------------------------------------------------------------
r9995 | tosterdahl | 2014-08-26 21:42:59 +0200 (Die, 26 Aug 2014) | 1 line

Added a 2500ms time-out waiting for a acknowledge. Added a 100ms time-out when using keypad id 0x70 (no acknowledge requrired). Timeouts according to spec.
------------------------------------------------------------------------

The workspace: svn+ssh://tosterdahl@svn.code.sf.net/p/openremote/code/workspace/toesterdahl/Controller_2_1_0_Integration

I guess a lost acknowledge is a indication of some other problem, but at least this fix makes the system more resilient to such an error. The entire point of adding the acknowledges and the semaphore is to make the system more responsive. Without acknowledge the controller will only respond after it's internal time-out of 100ms (for kepadid = 70). With acknowledge it should be possible to achieve faster responses. Most importantly messages does not get lost if they are repeated too quick.
