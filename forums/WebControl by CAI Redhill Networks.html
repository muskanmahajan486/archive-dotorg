Hello all,

I have been using the board called WebControl from CAI. This has 8 Input, 8 Output, 3 analog inputs, 1-wire temperature, humidity, internal 4000 lines of code capability, and is X10 compatible using 2 of the outputs and an RF transmitter. It is an exceptional value and can be had for around 30 bucks from Amazon or Ebay. I have boards 2 running.

I have installed OR controller, made a panel, and have it operating on my Android. I can turn on outputs on the Webcontrol board via 3G wireless, though internet, through my router, and on to my home LAN. So that is good.

What I can not figure out with OR is if I can retrieve and display data FROM Webcontrol or other web devices. For example, to receive the status of a single digital input, say DI2, I would type in my browser:

http://MYIPADDRESS:PORT/geti2.cgi This would return a "0" or "1"

If I want the status of the entire board I would type in my browser:

http://MYIPADDRESS:PORT/api/status.xml This would return a lengthy XML file with status of everything on the board.

Can OR retrieve this data? If so, how do I do it. I have searched the forum and documentation to no avail.

Thanks in advance,

Mike

Mike,
sounds like an interesting board. Do you have more information about it?

The first status call should be already possible with the HTTP protocol. Just create a HTTP protocol command and use the given URL. After that, create a custom sensor and link that to the new command. The custom sensor can return your on values (states) based on the returned string from the HTTP command.

The second call is not yet possible, but we are currently extending the HTTP protocol to support XPath expressions. This way you could parse the returned XML with XPath and give the XPath result to a sensor.

--Marcus
