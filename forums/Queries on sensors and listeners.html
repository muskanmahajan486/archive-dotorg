Posted by rwest at Jun 27, 2013 21:12

Glad that it's working for you.

You're right, I pointed you to the wrong designer, it's staging that has been updated with the Marantz protocol. I'll fix in the post above.

For adding commands, depending on how much development you want to do, you should be able to add some yourself.
Simple one way commands can be added by editing the marantz_avr_config.xml text file under WEB-INF/classes/org/openremote/controller/procotol/marantz_avr

  <command name="RETURN" class="org.openremote.controller.protocol.marantz_avr.commands.NoFeedbackCommand" value="MNRTN"/>

The above definition for instance, says that when the "RETURN" string is used as the command in the designer, it'll send the "NS" value to the receiver, and not wait for any answer.

  <command name="NETWORK_CONTROL" class="org.openremote.controller.protocol.marantz_avr.commands.NoFeedbackCommand" value="NS">
    <parameters>
      <parameter name="CURSOR UP">90</parameter>
      <parameter name="CURSOR DOWN">91</parameter>
      <parameter name="CURSOR LEFT">92</parameter>
      <parameter name="CURSOR RIGHT">93</parameter>
      <parameter name="ENTER">94</parameter>
      <parameter name="PLAY">9A</parameter>
      <parameter name="PAUSE">9B</parameter>
      <parameter name="STOP">9C</parameter>
      <parameter name="SKIP PLUS">9D</parameter>
      <parameter name="SKIP MINUS">9E</parameter>
    </parameters>
  </command>

Above definition takes parameters into account. So that if you have the "NETWORK_CONTROL" command in the designer, with the "ENTER" parameter, it'll send "NS94" to the receiver.

New commands won't be available from the designer though, so either list the required changes here and I'll see how we can update the staging designer, either edit the controller.xml file manually (but you would need to do this each time you change your design).
For parameters, it's a free text field with no validation, so it doesn't matter.

As for your help, we always appreciate users contributing to the project and love to see what people do with OR.
I did start a documentation page. If you find anything missing, typos, weird wording, ... please let me know, that would be great help.

Testing is also appreciated, if you can let me know what works and what doesn't, we can document that for others.

Thanks,
Eric
