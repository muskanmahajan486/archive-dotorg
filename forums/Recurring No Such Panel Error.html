Comments:

The message is caused by the web console; as part of it's initialisation procedure it checks whether the web console exists and whether it is secure/accessible. To do this it sends a REST call asking for a panel named ""; if that part of the REST API was secure then the web console gets an authorization required response otherwise it gets a no such panel response which is what you are seeing in the controller output.

There is nothing you can or should do about this; the controller is providing information (it could be argued that it is an information message not an error but the current code throws an exception).

Long term there is a plan to implement a REST API that reports the capabilities of the controller (is it secure etc.) but until then you'll have to ignore the messages.
