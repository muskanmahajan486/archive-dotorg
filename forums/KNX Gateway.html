Posted by schotte at Oct 15, 2010 21:52

You can do this through the Controller HTTP API although today it still requires that you program such logic yourself (we will add scripting engine bindings in the next major release which should help non-programmers accomplish similar tasks).

Saw in your other post that you compiled the iPhone client yourself, so maybe you're up to doing some HTTP/REST programming?

The relevant API is documented here:

  • Controller 2.0 API Documentation

In particular the sensor status request and sensor polling request (your incoming events from IR detector) and send write command for your http request.

At the moment you need to create your own JavaBean that puts these together (your logic) and configure that in the controller (Spring config). Or if Java is not your fancy, use any other language and use the HTTP API to pass commands and run your process next to the controller.
