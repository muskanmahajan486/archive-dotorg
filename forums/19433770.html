Yes, beehive is another building block and deployed as a seperate war file. Either in the same container or different container as the modeler. The designer is not completely decoupled from the database (yet) so it needs the REST URL for Beehive and also a JDBC DB URL. Both (Beehive and Modeler) have to use the same DB. The REST URL and DB URL are configured in config.properties WEB/classes folder.

The might be some documentation about this but I am not sure.
