For devices that actively broadcast their status information, you can implement an EventListener interface. There's no documentation for it yet beyond the Javadoc included in the source. You will need to implement a thread listening to the incoming data yourself and then push the values into the controller's state cache (using the sensor reference as shown in the example).

I expect some slight API changes to occur on this interface still but if you plan to contribute your implementation back to the OpenRemote project, I can probably deal with them myself as part of the regular maintenance cycle.
