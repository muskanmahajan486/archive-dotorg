So, I found from myself...

The mistake was coming from the fact that I used the same KNX group address for the STATUS, instead of using a separate group address, linked with the feedback object of the KNX_physical device in ETS.

My mistake was based on the tutorial http://www.openremote.org/display/docs/OpenRemote+2.0+How+To+-+KNX+Switches, where this is showed like that (same group address), what is wrong in this case.

I didn't write the openremote application... But with what I have seen, it seems to me that it's a must have in this case. It is very possible that any other application such KNXDroid (wich I don't know at all) is written by another way and work without any separate object.

You can always add in your installation a separate group for the feedback message via the ETS programm.
You should affect the feedback object of your device to this group. On some KNX devices, you need to activate these feedback objects in the parameters section.

If you can't find it, tell me what KNX device you have, and I will try to find it for you.
