Since my Home Controller runs on a flash drive, I'm concerned about excessive write cycles... and most commercial Home Automation applications don't care about this. I've found few information regarding this feature on OpenRemote so here are my questions:

1. Which data base does OpenRemote controller use? MySQL, SQLite, ...?
2. Is "live" data (current values and status) stored in data base too?
3. Is "live" data (current values and status) stored in hard disk or RAM?
4. Is there any reference implementation of OpenRemote controller for any flash-based computer?

Thanks again and sorry for my noob questions.

Daniel.

Comments:

1. Currently no database
2. Not at the moment (RRD has been suggested a couple of times)
3. No.
4. Alix OpenRemote Reference Implementation - Hardware Bill of Materials, various versions of Sheeva-plug, Guru-plug, etc.

If you're worried about memory wear, one area you want to tune is Tomcat's logging which can be quite verbose (you can configure Tomcat's logging in /conf/logging.properties and the controller app's logging in WEB-INF/classes/log4j.properties). Log writes also tend to be slow on flash drives.

If you do need more logging, you can also look at configuring log writes to RAM instead of flash, if you have enough to spare.

More info here: http://pcengines.ch/cfperf.htm
