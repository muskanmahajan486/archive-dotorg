Posted by jalyst at Mar 07, 2013 15:08

So the free version isn't a few revisions behind or less functional, it's just geared so that it's ideal for Pro Installers? (hence all the hosted stuff & support)

There are some slight differences – for example with Z-Wave where we cannot release the implementation to Open Source. The releases are not synced either so some features may appear first on Professional version or vice versa, depending on availability of anyone to do the release (at least until we can get someone dedicated for release management). Some import functionality may also only be available on professional versions where repeated tasks over multiple accounts would otherwise get tedious.

By and large we attempt to make it clear on the wiki documentation whether some features are not included in the DIY version.

Is there a commercial version targeted for DIY'ers? (who may or may not dip their toe into the Pro Installer scene longer-term)

There's no separate product per se, but nothing in the professional version makes it unsuitable for DIY users. The free version may require some extra work to be done (in a DIY sense) where as the pro may have added convenience, or if you get the hardware, pre-installed controller and easier set up.

In that case, what's the recommended controller hw to go for?

Most PC's running Linux, Windows or Mac will do as long as there's sufficient memory (would recommend 512MB although minimum is lower) and a CPU that's at Intel Atom level (the low-end) or above. ALIX hardware has been pretty well tested and works, and DMP ebox is obviously what we test as well which you can go and purchase by yourself too.

There are other hardware platforms that you see people use, and they get mentioned here on the forums regularly.

Some recommendations on; protocol bridging devices, panel(s), & end-devices to start with, would be really appreciated too.

I can't recommend anything without knowing what you want to achieve, what your budget is, what's the availability of devices in your region, etc.

If I go for a commercial version, can I run it on a distro of my choice or is there a preferred distro/env?

The regular Fedora, Ubuntu distros are ok, as are some specialized distributions such as Voyage. Basically anything either from the fedora or debian line should be ok. The key requirement is a modern Java VM (OpenJDK for example) on that distribution. Java is usually available on most x86 architectures, if you go to ARM then the choices are fewer.
