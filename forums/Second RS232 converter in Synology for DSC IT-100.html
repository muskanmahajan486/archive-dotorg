For the Serial protocol it is no problem, since you define the port with every new command. DenonSerialAVRProtocol and the RussoundProtocol currently can not cope with multiple ports. Principally the issue is that in this case there is only one global setting for the port, so you can not define more than one. One could also imagine a setup where you want to connect to a device on one port over more than one Protocol. There is a concept for a shared management of ports across protocol implementations, but it is not implemented through-out.
