Hi guys,

I was playing arround with the ZWave support in OR under Linux when I found the following issue. It seems like the zwave.jar uses the java rxtx library which has one little problem. As long as the serial port you try to open is in the form of /dev/ttyUSBX everything works fine and the port is properly opened. However, it is convenient to assign a symbolink link to the device (for example /dev/zwave) on startup when having multiple USB devices connected to the PC. In my case the symbolink link is created but the librxtx cannot open the device.
It took me some time before I found out that this is actually a known problem of the java rxtx library. http://www.raspberrypi.org/phpBB3/viewtopic.php?f=5&t=6011

They suggest to add the following lines to the code before opening the code:
String SerialPortID = "/dev/ttyAMA0";
System.setProperty("gnu.io.rxtx.SerialPorts", SerialPortID);

Could you do me a favor and replace the serial port string with the one configured in the controller.xml and add those lines to the zwave.jar? I think more people will run into the same issue.

Thanks a lot.
Alex
