Hi Nir,

Yes macros only offer you delays and sequences of commands what you need is rules. Search the forums for "rules" and/or "drools" and you should find some examples.

The KNX protocol sends/reads to the various group addresses and I don't know off the top of my head whether or not all the DPT (Data Point Types) you will need have been added to the KNX protocol but again the forums should help there.

I have a KNX installation that uses an outdoor Merten temp/light sensor and have it setup to open a window if the temperature is above a certain threshold and it also activates a panel of smart glass when the light level exceeds a threshold. All the logic for this has been implemented in the KNX modules (InZennio actuators have logic functions built in). I prefer to keep as much KNX logic inside the KNX network but there are limits to what you can do without buying expensive logic units.

As far as I have seen from the documentation of the OR KNX Command name, it looks like the OR does not support all DPT.
It does not support 1.002 (Boolean), 1.003 (Enable), 1.008 (Up/Down), 1.009 (Open/Close), 1.010 (Start/Stop). These are all 1 bit.

In practice on/off, up/down, start/stop, true/false, open/close are all the same frames on the knx bus (1-bit) so you can simply use the on/off for all of them, the result is the same. The DPT, for some odd reason, is not actually encoded into the frames itself.
