Posted by craigansell43 at May 16, 2013 22:25

This is because of the apostrophe at the end of ./openremote.sh run'

It doesn't belong there, and is a special character with the shell terminal which will cause the appearance of the > sign.

So try without:

./openremote.sh run

If you get an error about permissions, then inside your 'bin' directory, type:

chmod a+x *
