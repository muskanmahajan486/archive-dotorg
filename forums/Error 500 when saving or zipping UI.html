Forums : Error 500 when saving or zipping UI
This page last changed on Apr 26, 2013 by slong.

Error 500 is generated in the designer when trying to save or zip a UI design with any elements added.
SOLVED - the left and top attributes are shown as floats when the element is dropped or dragged using the mouse. Changing these two values to integers allow the saving to complete successfully.
My configuration is Chromium 27.0 on OpenSUSE 12.2.9
