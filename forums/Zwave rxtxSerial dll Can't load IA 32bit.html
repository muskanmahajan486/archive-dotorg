Posted by crossbred900 at May 30, 2013 17:46

Normally that warning is not a big issue – the Java files (JAR) have not been updated with the same version information as the compiled native binaries (DLL). If it is just an omission to update the version information but not a real API compatibility issue then the warning won't impact you – this is the case with the original RXTX binaries for example, the maintainer forgot to update the version information.

You can try to update the JAR too – there's of course a small chance of it creating issues since it hasn't been tested so you can go either way. Live with the warning or try to get rid of it and hope there wasn't any other issues with the new JAR archive.

If you choose to update the JAR, then that will go into OPENREMOTE/webapps/controller/WEB-INF/lib directory – replace the existing RXTX 2.2 jar.
