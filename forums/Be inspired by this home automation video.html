Posted by mredeker at Aug 02, 2012 09:00

As Marcus said, most of it is doable, I would say that the only thing I've seen that's not here right now is the voice control.
Siri won't work as there is no API from Apple to integrate with it. But there are several libraries for iOS developer to provide speech recognition on the device (one of them being Open Source).
I looked briefly at including that in our iOS console but this really was not a priority, so skipped to more useful things.
But if some iOS developers want to have a go at it, I'll be glad to provide support.

For the "proximity trick", I did once implement as a demo / proof of concept something similar, though with limited range. I used a small tool on the Mac that did listen for bluetooth devices and could trigger AppleScript when it detected a device connecting or disconnecting. From these scripts, I was issuing a REST call to the OpenRemote controller. So when my Mac detected my iPhone near it, it would trigger a macro in OR (and would trigger another one when iPhone went away from Mac).

If you find some kind of "RFID" device that can detect your car arriving near your house, you could already either have a protocol for it in OR, this could toggle a sensor value and then based on rules you can do whatever you want.

For video surveillance, if your cameras provide a MJPEG stream, you can use the Web element in OR and have your video displayed in the consoles already.

At this stage, it still requires a lot of imagination and tricks to do what you want, but we're working on making things easier.
Posted by ebariaux at Aug 02, 2012 09:43

Even the SIRI part can be acomplished at home.
SIRI is contacting an APPLE server for each request and you can intercept this request if you map the server address within your LAN to a machine in your network.
That machine needs to run the SIRI prox which can be used to sent REST calls to OpenRemote.

So, something like "Computer, please turn living room light on" is possible.
