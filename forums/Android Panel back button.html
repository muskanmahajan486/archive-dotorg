Using the designer, is there a way to map the 'hard' android back button to an open remote action?

The reason I ask is because I keep pressing the button to go 'back' on the remote (to take me back a level) but Open Remote exits the app.

Most android apps allow the use of the back button within the context of the application. It feels very counter intuitive to have a 'soft' button for the back (up a level) action.

Thoughts?

Comments:

There's nothing in the designer yet that allows command mappings to hard buttons. To make it work in a generic way, we need to add mechanisms to designer to recognize what device type and operating system the panel design targets. This is something we've discussed before as a needed feature.

In the meantime, a short cut to alleviate the issue is to assign some default action to the back button that works universally (other than exit), for example going back to previous screen. This may be an acceptable solution if it works for all Android panels equally. If that's the case, would appreciate patch submission to the Android codebase.
