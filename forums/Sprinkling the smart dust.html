Posted by llauren at Dec 05, 2011 21:20

It depends.

For function definitions (calculations, etc.) Java syntax is required with rules. Simple actions based on events can be handled with decision tables. The first iterations you will see will default to the Java-like rule language syntax (the rules are compiled into Java). In most common cases the rule consequences (the Java syntax) are one-liners.

The service provider interface where rules are plugged in can be used to introduce imperative scripting languages instead of rules. None are provided yet since rules do cover most of the common use cases. If you want to contribute one, you'll need to re-learn Java in order to provide the script language integration to the controller. There's no documentation yet how to do this so you will also need to be able to read the Java code to figure out how to do it.
