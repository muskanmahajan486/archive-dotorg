Comments:

There's a new iOS app in the pipeline for distribution (once I can figure out what's going wrong with signing the app for approval process) although this won't have iPhone5 resolution support per se (although will have fixes for things that were broken in iOS6), as far as I understand.

What's New in this Version

The slider implementation has been changed in this version to make it work as desired on iOS 6. It is being standardised to look and behave identically on all OpenRemote console, as defined at Slider.

This means that sliders using custom images, that looked OK before might render incorrectly. Please update your design taking the specifications of the OpenRemote slider widget into account for correct rendering.

Bug fixes:

  • If the same sensor id is used multiple times on the same page, it appears multiple times in the poll query
  • Even when single screen in group, pagination gesture is possible and crashes the app
  • In group with multiple page, scroll to page prior to first crashes app
  • Sensor value not received correctly
  • iOS console crashes on launch if can't connect to controller
  • User authentication not taken into account
  • Controller not found message when using remote proxy
  • Min/max track image display on sliders do not work under iOS6
  • Timeout downloading the images on bigger configurations
  • Error downloading panel configuration
  • Old message about auto-discovery is still present in UI
  • Back button does not return to previous screen

The issue you're seeing with audio is because the current 2.1 release in the AppStore had some VoIP/SIP librairies included (it was a test and they should not have made it to the Store). Those take over the audio system in some way.

As Juha said, the coming release 2.1.1 will have this fixed + everything in the release notes.

The next planned version 2.2 will have additional fixes but not iPhone 5 screen resolution support yet. This require some more re-work of old code (and will also mean we drop support for anything earlier than iOS 4.3).

But I know this is important and we'll work on it as soon as resources / planning allows.
