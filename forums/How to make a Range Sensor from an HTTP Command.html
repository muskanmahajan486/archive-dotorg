I think I found a work around for a range sensor from an http call. It works on a slider that I have based on this sensor. In this example I do call an element from the XML file below

By using the XPATH expression number(concat(//response/gauge_power,"")) the string content of gauge_power is transformed in the numerical value 14.2. For reasons unclear to me as yet, I had to concatenate the retreived string value with an empty string before the number conversion would work.

In my own application for variable energy_today I added *1000 at the end of the Xpath expression to convert this variable from kWh in Wh. In doing that, I got rid of the decimal point.

Looking for other problems I found this error in the logs:

WARN 2013-02-20 11:56:15,567 : Sensor 'SolarDayProduction' (ID = 100,247) is RANGE type but produced a value that is not  an integer : '4011.9999999999995'

This number does not show up in the user interface. To make things formally correct, I should have used round instead of number in my XPATH expression:

round(concat(//response/gauge_power,""))

After this change I haven't seen the errors again
