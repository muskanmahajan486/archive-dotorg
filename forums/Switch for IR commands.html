Hi all,

I'm trying to create a switch-button for muting the sound. I would like the following behaviour:

  • pressed once, it send out the 'mute' IR command (HTTP through an Itach IP2IR) and the button turns to the MUTE ON state
  • pressed again, it sends out the same 'mute' command (which unmutes the receiver) and the button turns back to the MUTE OFF state

I couldn't get this to work, is it only possible to change the state of the switch by receiving some data through a sensor?

Comments:

Yes, changing the switch state with a sensor is possible. You need a "Switch" sensor and a new command that returns the state (on or off).
The problem via infrared is, that you don't receive feedback and it might be complecated to create a command which can determin the current state.

I had a similar situation on my X10 switches which did not provide feedback. The workaround I did was to use In-Memory Virtual Command along with rules. See the link below where Eric explained
