Comments:

The modeler_tmp directory is indeed just a cache and will get refreshed.

The resources/userid is the place to make the changes and more specifically the openremote.zip file.
This is the place where changes need to be made.

Please make a backup before trying this but I would:

  • delete a content of modeler_tmp
  • delete all files from resources/userid EXCEPT for openremote.zip
  • extract openremote.zip to a temporary folder, do the changes there (delete non used images, update images you want to change) , the re-package openremote.zip and put it back in resources/userid
