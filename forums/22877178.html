I think there was some discussions earlier on how to install unpackaged versions on Synology but I can't recall right now where I saw them – or what to search for.

In very simple terms, the OpenRemote.zip can be unpacked (unzipped) on a file system and runs as-is from bin/openremote.sh as long as Java VM has been installed on the system. The packaging I suppose takes care of things like auto-starting and so on. The actual scripts for the package are available on patters' website as far as I recall.

So if you can locate a webapps/controller directory in your existing install, replacing that directory from the downloaded OR zip should go a long way. To be completely accurate, you may want to replace the logs and bin directories too just to be safe (or in some cases the conf directory). But mainly all the code changes will be in webapps/controller directory – some configuration changes (which the code may rely on) may appear in the other directories mentioned.

That's probably still easier than installing OR on a vanilla Tomcat – it'll work but it may assume the Tomcat configuration has been set in a specific ways.

On the OpenRemote 2.0 How To - Synology with Z-Wave page there's at least partial instructions on how to install an alternative version of OR on Synology. Although this is in context of Z-wave, I think at least parts of it are relevant to you and may help you get started:

OpenRemote Software Install

1) Install Synology package for Open Remote: Synology Package

2) Follow the instructions under the Configure of System Section.

Since you already installed the OpenRemote package in item (1), you just need to:

  • Ensure you can get snapshot OpenRemote distro onto the Synology. The easiest way is to copy from windows to Synology share. Then you can access under /volume1 folder.
  • Unzip the snapshot Openremote distro using the "unzip" command on the Synology: 
    > unzip /volume1/download/OpenRemote-Controller-2.1.0_Developer-Snapshot-20120920.zip
  • Remove the webapps folder under the OpenRemote package install 
    > rm -fr /volume1/@appstore/OpenRemote/webapps
  • Copy snapshot distro webapps directory into package folder 
    > cp /volume1/download/OpenRemote/webapps /volume1/@appstore/OpenRemote
  • Ensure permissions are set correctly 
    > chown -R openremote.users /volume1/@appstore/OpenRemote/webapps

3) OpenRemote uses a Java library to communicate via serial port to your Aeon device. For this, it uses the native libraries stored in: /volume1/@appstore/OpenRemote/webapps/controller/WEB-INF/lib/native.

Unfortunately, the distro for OpenRemote assumes it's running on an Intel processor. For Synology, this will not work as it's using a RISC processor. So, you have to retrieve the ARM compiled native serial libraries.

For this:

  • Download the librxtxSerial-2.1-7.so for ARMv5 based Plug Computers from Oracle Blog and copy into /volume1/@appstore/OpenRemote/webapps/controller/WEB-INF/lib/native
  • Now that you have ARM based RXTX library installed, you need to link to this library from the OpenRemote native directory: 
      > cd /volume1/@appstore/OpenRemote/webapps/controller/WEB-INF/lib/native
  > rm -fr librxtxSerial.so
  > ln -s librxtxSerial-2.1-7.so librxtxSerial.so

4) Permission OpenRemote service account to allow the creation of OS locks. This will be needed to ensure a resource lock is created when communicating the serial port of the USB device.

For this:

  • Add a new group using the standard add group admin screen in synology. Review section 3 of How to create user accounts and groups on Synology NAS. Add a group called: uucp
  • Add the openremote user to the newly created uucp group
  • Verify that the group is setup correctly in a shell session: 
    > more /etc/group
    You should see something like: 
    uucp:x:65548:openremote
  • Ensure new group can access write to /var/lock folder: 
    > chown -R root.uucp /var/lock

5) Permission openremote to the USB port device.

  • Add a new group using the standard add group admin screen in Synology. Review section 3 of How to create user accounts and groups on Synology NAS. Add a group called: dialout
  • Add the openremote user to the newly created dialout group
  • Verify that the group is setup correctly in a shell session: 
    > more /etc/group
    You should see something like: 
    dialout:x:65549:openremote
  • Ensure new group can access write to /dev/ttyUSB0 device: 
    > chown -R root.dialout /dev/ttyUSB0
    The example here uses ttyUSB0, but the device assigned is what you'll observe in step (6) in the Installation of Linux Serial Drivers.

The latter parts 3-5 are related to using USB sticks in case of Z-wave. Not necessary for KNX-IP.
