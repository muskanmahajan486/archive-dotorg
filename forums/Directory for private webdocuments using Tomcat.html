Hi Pieter,

Creating your own custom HTML/Servlet content is very user specific and most users won't be requiring this

Using your own directory name within the webapps folder is fine OR uses controller and webconsole.

If you want to host your own HTML content and not have it affected by controller upgrades then you should be looking to have a separate web server running (another instance of Tomcat if you need Java Servlets or a more primitive web server for serving static HTML), this should then be run on a different port number.

I asked because some packages I have provide a private space, which is retained at updates.
I prefer to keep things together. I use this space for rather small pages used within OR. For example an xml file I retrieve from my solar panels. I did notice that data collected from the panels directly via the http protocol were a bit unstable. So I buffer it in that space.
As a side note: I started using Microsoft IIS server on my XP machine to serve these pages. I got the impression that Tomcat is more responsive.
