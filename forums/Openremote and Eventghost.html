Comments:

The UDP command implementation does not support sensors yet. Only TCP has some support for this.
Steps to get this working would be:
1) create one command which sends TCP todo some action
2) link this command to a button

3) create one command which sends TCP to receive some payload
4) create a custom sensor and link the command which is receiving the payload
5) use a label and link to the custom sensor to display the payload

You don't necessary need the "action" command and the button. Only the status command and the sensor is enough if you just want to display information.
