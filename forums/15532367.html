In the last month or so i've done a project that was about running OpenRemote Controller on ARMv5 Devices. It is requested that I share this information.

2 Goals were defined. Running OpenRemote Controller on the Guru Plug. And the Insignia infocast

Both are ARMv5 platforms, and are able to run an armv5 version of linux. On the guru plug this is already available, the insignia infocast runs a closed OS, details on how to build angstrom and get it on the infocast to prepare it for running the openremote controller are found hereand here

Running openremote controller is now possible with the headless edition edition of the Java Standard Edition.
Download it from http://www.oracle.com/technetwork/java/javase/downloads/embedded-jsp-135769.html
Take the ARMv5 Linux - Headless link, fill in the survey and you will get the link to the download page.

From there it is a case of transferring the tar.gz file to your platform (Plug Computer/Infocast or other armv5 platform) and installing it by moving all the needed folders to your root directory (usr and bin from the package to / )
*Java takes up quite a bit of space, about 60Mb. make sure you have the space on the root of your target.

After putting the tar file on your target it is a case of doing.

tar -zxvf <java-archive-filename>.tar.gz
cd java-archive-folder
mv bin /
mv usr /

Java should be working now, you can test this by trying
java -v

this should output the version of your java. If this is working you can go on and install OpenRemote controller.

Download openremote, Extract, Change to OpenRemote '/bin' directory and execute './openremote.sh run'
(check the permissions of openremote.sh, make it executable by chmod +x openremote.sh)

When this all works, you have the openremote controller on a low powered device. If you run it on the infocast, you will even have it on a low powered device with touchscreen and browser. Very nice when the Webcontroller will be released
