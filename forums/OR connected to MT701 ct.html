Hi Bruno,

I am not familiar with Berker MT701 but assuming it is a stand-alone IP device (with its own IP address and port that you can use to control it) then it's possible that OR can be used. Details depend a bit on what kind of protocol the unit offers over IP, we currently can support unidirectional protocol integration to IP devices.

While there's no web user interface available yet, it is trivial to test this through the HTTP/REST interface.

Download the controller: http://archive.openremote.org/download/OpenRemote-Controller-1.0.0_Beta6_JavaOnly.zip

Unzip it and deploy the following file in OpenRemote-Controller-1.0.0_Beta6_JavaOnly/webapps/controller/controller.xml


<?xml version="1.0" encoding="UTF-8"?>
<openremote xmlns="http://www.openremote.org" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
xsi:schemaLocation="http://www.openremote.org http://www.openremote.org/schemas/controller-1.0-M3.xsd">

  <buttons>
    <button id="3">		
	<event>1</event>
    </button>
  </buttons>

  <events>
    <irEvents></irEvents>

    <knxEvents></knxEvents>

    <x10Events></x10Events>

    <httpEvents></httpEvents>

    <socketEvents>	
      <tcpSocketEvent id="1" label="Some Label" ip="1.1.1.1" port="1111" command="something here"/>
    </socketEvents>

    <telnetEvents></telnetEvents>
  </events>
</openremote>

Change the IP, port and command settings in the TCP socket event in controller.xml.

Start the controller from OpenRemote-Controller-1.0.0_Beta6_JavaOnly/bin/startup.sh|bat. Then on your web browser invoke the following URL:

http://localhost:8080/controller/rest/button/3/click

Let us know if you get any luck with it.
