Posted by marcf at Nov 11, 2010 18:33

OpenRemote KNX has the same issue, the restriction is on the gateway, not on the software:

I can only connect with one iPhone to my KNX installation: typical gateways from Siemens or ABB accept only one connection at a time. If you've a Linux server around your house, you can use the eibnetmux daemon. It acts as a multiplexer and allows lots of connections. It also supports user restrictions.

The difference being that controller is always on, OpenRemote KNX connection is only on while your app is on.

There are couple of ways to deal with this but multiplexing is the best way to do it.

Posted by mb at Nov 16, 2010 10:24

my Server linux with Eibnetmux is in a different subnet, across a ROUTER; do you think is a problem with autodiscovery detection ?

Yes. That would be your issue. The multicast is unlikely to bridge subnets unless you explicitly configure the router to allow it.

The best setup would be the eibnetmux and controller in the same subnet, and you KNX gateway in the other subnet (if possible, and eibnetmux is ok with this setup by specifying fixed IP to cross the router).

So if controller/eibnetmux are in their own subnet, and knx gateway is on another subnet, discovery should work without conflict. Let me know if its possible for you to reconfigure this?
