Comments:

How did you create the controller and panel.xml ? Did you edit them manually or used the modeler from a branch ?
If you did edit them manually, get rid of the web tag from the controller.xml file. It is not really required and as it's not in the schema, XML validation fails, hence the invalid controller.xml error.

Another solution, which works irrelevant of whether you created files manually or through modeler is to update the controller schema to add support for the web tag.

Martin,

the <web> element is not supported from the designer deployed at: http://composer.openremote.org/demo/

Here are a panel.xml and controller.xml which worked for me. Maybe you can use them as a template.

controller.xml
=============
<?xml version="1.0" encoding="UTF-8"?>
<openremote xmlns="http://www.openremote.org" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://www.openremote.org http://www.openremote.org/schemas/controller.xsd">
<components />
<sensors/>
<commands/>
<config/>
</openremote>

panel.xml
=============
<?xml version="1.0" encoding="UTF-8"?>
<openremote xmlns="http://www.openremote.org" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://www.openremote.org http://www.openremote.org/schemas/panel.xsd">
<panels>
<panel id="23" name="panel2">
<include type="group" ref="24" />
</panel>
</panels>
<screens>
<screen id="25" name="Starting Screen">
<absolute left="0" top="0" width="320" height="480">
<web id="4501" src="http://www.openremote.org/download/attachments/12845601/Tablet2_288x373.png"/>
</absolute>
</screen>
</screens>
<groups>
<group id="24" name="Default Group">
<include type="screen" ref="25" />
</group>
</groups>
</openremote>
