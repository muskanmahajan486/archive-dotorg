Sometimes my controller does not start properly and I can detect this from rules. In such case I need to restart the controller which I do by touching the controller.xml via shell ex command. This is rather a rough hack, so is there a better way of restarting the controller?

Comments:

Touching controller.xml is a valid way of triggering a soft-restart (re-deployment) of your controller configuration. It is not a reboot of the controller per se (hard reboot) as it doesn't restart the virtual machine.
