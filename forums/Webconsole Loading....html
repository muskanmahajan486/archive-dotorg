Yann in the process of fixing the problems you were seeing I've noticed some differences between the way the various consoles render: -

  • For buttons Android console will display image and label where as iOS and Web Console will only show label if the image doesn't load (not sure what the 'correct' behaviour is)
  • For widgets in a grid panel the Web Console fits the widget to the grid cell, Android and iOS do the same except for image widgets where if the image is smaller than the cell then the image size is used instead (as a result the Web Console currently stretches the image to fit the cell which makes your screens look strange).

These differences in rendering behaviour need to be clarified but I will change the Web Console image rendering to match iOS and Android.

Hi Martin,

Some support is needed in the Web Console to let you know why it can't connect to the Controller you want to connect to.

If you cannot get past the Controller List screen then my guess would be that you used the Controller Search button to find controllers on the same LAN as the Tomcat server running the WebConsole but you are outside of the LAN? This is an issue because controller discovery returns the internal LAN IP of the controller but if your outside of the LAN then you need to be using the external IP (or DNS name) of your LAN.
