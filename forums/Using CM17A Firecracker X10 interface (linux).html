The Wireless CM17A (serial) "Firecracker" interface is not yet functional with OpenRemote, but it can be made to work reliably using the shell execution protocol to invoke the BottleRocket command line interface for this device. This specific method is for linux-based OR controllers.

1) Install, configure, and test BottleRocket.

2) Create a device for the CM17A:

3) Create a command for each X10 device in your system. You'll need a command for ON, OFF etc. Of course use the path where br is installed on your machine. The command parameter corresponds to the specific action you wish to perform.

4) You can now assign your commands to controls using the UI designer.

Hope this helps someone.
