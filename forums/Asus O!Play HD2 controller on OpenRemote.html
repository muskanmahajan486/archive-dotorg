Hello everyone!

I have sucessfully implemented the Asus O!Play HD2 media player control on OpenRemote. The media player itself has a web control for smartphones, but I wanted to have the control integrated on my OpenRemote app (this way I can control my KNX installation and the media player on the same app), so if someone wants to control the media player over the network, here it is.

I believe that the cgi commands are the same for older models, like O!Play R1, R3, etc, but feel free to try it. Tested and working on a O!Play HD2.

How-to:

Create a New command for each button, choose HTTP protocol, and place the correspondent URL for each button:

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=return

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=left

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=right

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=setup

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=power

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=mute

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=pause

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=play

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=stop

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=up

http://oplayba/www/cgi-bin/IpodCGI.cgi?id=0&command=down

Haven't found any cgi commands for volume control yet, I don't know if they exist.

Cheers,

UPDATE 1 Please change 'oplayba' in the url to your media player IP address. Sometimes the 'oplayba' doesn't seem to work very well.

Found out that the 'oplayba' network name would only work if you would start the SAMBA server and then stop the service when returning to the main menu. I believe that the main reason for this to happen is for the fact that the O!Play doesn't allow to have SAMBA server running while whatching a movie or listening to music. It only allows one service at a time, either multimedia features, either FTP server, either SAMBA server. So, please don't use 'oplayba' network name while using this OpenRemote integration, use the player IP address, it's much safer.

E.g.: http://192.168.1.149/www/cgi-bin/IpodCGI.cgi?id=0&command=left

UPDATE 2 New commands available

Display key: http://PLAYER_ADDRESS/www/cgi-bin/IpodCGI.cgi?id=0&command=display

Option key: http://PLAYER_ADDRESS/www/cgi-bin/IpodCGI.cgi?id=0&command=option

Next key: http://PLAYER_ADDRESS/www/cgi-bin/IpodCGI.cgi?id=0&command=next

Previous key: http://PLAYER_ADDRESS/www/cgi-bin/IpodCGI.cgi?id=0&command=prev

Repeat key: http://PLAYER_ADDRESS/www/cgi-bin/IpodCGI.cgi?id=0&command=repeat

Caption key: http://PLAYER_ADDRESS/www/cgi-bin/IpodCGI.cgi?id=0&command=subtitle

Language key: http://PLAYER_ADDRESS/www/cgi-bin/IpodCGI.cgi?id=0&command=audio
