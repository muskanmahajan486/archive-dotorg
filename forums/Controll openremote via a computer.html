I like the ability to control KNX via a phone. However for the moment my primary concern is controlling the KNX-system via the computer (I don't have any KNX-equipment at the moment, so I am checking the software beforehand)

What i want to do is

1. Send KNX-telegrams from the computer with openremote installed, without going through a phone.
2. Receive info from the KNX-system, for instance a temperature, which i can read with another program
3. As an alternative to 2, an event based system "if its 3 o-clock, and the temperature is above 22 degrees, turn on the light".

Is any of this possible with openremote?

Comments:

1. Yes, if you want to have a GUI on the computer, there is a Web console that can be used.
If you don't want a GUI, there is a REST API that can be used to trigger commands (or read sensors).
e.g. on a Linux/Mac system, you could use something like 

curl http://localhost:8080/controller/rest/control/51/click
.
See documentation of REST API here

2. Yes, using REST API.

3. Yes, this is also possible using the rules engine.
