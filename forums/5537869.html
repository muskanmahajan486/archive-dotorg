Posted by handy.wang at Nov 06, 2009 05:30

First you have to import the ets application for your IP-Interface into ets. If you go to the settings of the device you will probably see that you can either give the device a fixed IP or set it to use DHCP. If it is not configured for DHCP it will not send the DHCP request to your router. Now you have to use ets to program the device with the settings.

To program bus devices with ets you have two options: local or bus. Local is used to program the bus interface (RS232, USB, IP depending on your hardware) which is directly connected to the computer ('locally') and bus is used to program all other bus devices (actuator, sensor) which are only connected to the computer through the bus. If you cannot program your IP-Interface through the local setting (computer and ip-interface connected with a ethernat cable an configured for the same subnet) you need another bus device (eg. usb gateway). Then the computer will be connected with the usb device to the bus and the IP-Interface will be connected with the bus.

The USB gateway is just another way of accessing the bus. It has nothing todo with you ip interface.
