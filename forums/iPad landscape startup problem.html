Using the development build of the iPad app, I'm seeing a problem with a landscape screen on initial start up. I have only landscape screens designed for the iPad so the screen that loads on start up has a landscape orientation. Sometimes when I start the app, the start screen loads, briefly, in portrait mode then rotates to landscape mode. However, after this rotation, the screen is not displayed correctly. It appears to offset towards lower left corner so that what should be the lower left corner of the screen is not visible. The entire tab menu is not visible nor are things that would normally appear on the left and bottom sides of the screen. Rotating the iPad 180 degrees so that the screen is redrawn fixes the screen and everything is fine after that.

This doesn't always happen on start up. But, about 25% of the time I can force it to happen by going into the settings, turning Auto Discovery off and then back on, and then selecting the iPad Panel Identity.
