Yes, thats the one. Here follow one tip which I'm using to solve the problem which returning of timestamp was introduced in the first place.

This is done because IRtrans (that is where I use it for) is always sending the remote name and the button name. This means the received sensor value is always the same and since "old" value and "new" value are the same, the rules will not trigger when I press the same remote button a second time.

I was facing similar problem when trying to bind functions to double click of EnOcean buttons. In fact the solution is quite simple, just follow the template in a rule which should react on multiply clicks:

rule "button click"
salience -100
when
  $b: Event(source=="xyz")
then
  // do whatever you want with the click
  retract($b); // This allows for consecutive clicks to trigger this rule again
end
