Posted by dnoguer at Jan 07, 2013 10:15

There is no step-by-step as far as I know
We know the documentation still has to be improved

Here is a quick wrap-up:
1) OpenRemote consists of three parts

  • Controller software (Brain of the system. Has to be installed on a machine and communiates with the devices)
  • Online designer (Is used to configure your commands which tell the controller how to talk to your devices and to create your panel layouts which tells the console how your UI should look like)
  • Console (iOS, Android, Webb) connects to your controller to receive UI information and communicates with controller to display sensor values or execute actions

2) Install controller and make sure it runs (connect to http://<ip>:8080/controller via a browser)
3) Create a user account with the designer, log-in and create a design
4) Sync your design with the controller (Go to http://<ip>:8080/controller, enter your designer username and password and perform sync online)
5) Install OpenRemote console for iOS or Android or use webconsole
6) If the console discovers your controller (iPhone has to be connected to WLAN and on same network then controller), select controlelr and then select panel (this was designed in step 3)
7) If the console does not discover your controller add the controller manually
