Posted by mredeker at Jul 19, 2012 06:40

Did some testing
You need to use the workload field.
Else you get this:

GET /?TL=1 HTTP/1.1
Host: 192.168.2.210
Connection: Keep-Alive
User-Agent: Apache-HttpClient/4.0.1 (java 1.5)

If the workload field is empty the command the controller make it a a get statement instead of a POST.
Posted by kjkoopman at Jul 19, 2012 22:53

Ok, I just double checked myself and you have to put the "TL=1" into the workload field.
The HttpClient library is creating a request which contains 2 segments, one with the header and one with the data.
This is indicated by the "Expect: 100-Continue" header and follows the HTTP specification.

I was able to verify this with Wireshark (see attached screenshot):

You will have to make your Arduino board to understand that request.
