You could try the "ShellExe" command and have that perform a ping with count 1 towards your PC.

You will have to play around with the regex to filter the correct answer.
Negative ping should bring something like this: 1 packets transmitted, 0 packets received, 100.0% packet loss
Positive ping should bring something like this: 1 packets transmitted, 1 packets received, 0.0% packet loss

If you extract the 0 or 1 from "packets received" you can use that in a custom sensor to return "on" or "off".
This is just an idea and I did not test this.
