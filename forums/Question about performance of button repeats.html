I think the XBMC may have several different APIs to connect to it. I haven't tried it myself but do recall other people here on the forums mentioning TCP for example.

HTTP requests can be heavy to set up (build up and tear the connection between each request) compared to TCP connection or UDP packets. That might affect repeat performance. But as I said, I'm speculating I haven't tried XBMC yet.

From your description it sounds like the XBMC app might be using UDP packets to communicate, instead of HTTP.
