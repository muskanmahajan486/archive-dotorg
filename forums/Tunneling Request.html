Comments:

Not sure if 6 bytes for connection header is correct, if you count the bytes that are shown in KNX spec examples the connection header pretty clearly is 4 bytes (octets) long in all examples.

It seems inconsistent in KNX spec example pages where it sometimes claims 6 bytes as connection header length (which is the length of the EIBnet IP header, not the connection header) and then in a following example correctly has 4 byte length for connection header again.

So if it is indeed an error in the spec (which I suspect it is since the connection header structure images all show only 4 octets) then that error would propagate rather wide coming from the original specification error (including some of our wiki pages, it seems).
