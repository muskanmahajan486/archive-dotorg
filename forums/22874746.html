The switch gets its value from a read command. Depending on the protocol, this might be an asynchronous notification coming from the device (e.g. update coming from a KNX bus) or a poll (in your case, explicit read command to CBus every n seconds).

One possible solution to make the UI more responsive is to use virtual commands to have in-memory values linked to the switch.
Then use rules to the cbus devices based on the in-memory values.
You can also set-up sensors that do poll the devices at longer interval. Again using rules, based on those sensor values, make sure the in-memory values stay in sync with the physical state.

There are a few examples of using virtual commands and rules with switches in the forums.
