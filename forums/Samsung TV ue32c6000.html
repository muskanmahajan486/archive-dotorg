The code in OpenRemote is from the SamyGo implementation and is only compatible I believe with the Series C and D TV's... (I think) if this was the source, im getting the same error as above.

Looks from the source the port is hard configured to 55000

"private final static int SAMSUNG_TV_REMOTE_CONTROL_PORT = 55000;"

From what i can see in the latest dev documents on Samsungs site the latest version listen on Port 80 to initialise the Samsung device. The Samsung wants your TV to confirm that you trust what is connecting and then it will store your devices Mac Address in your TV so that you can communicate with the TV and send commands.

Setting up communication to a Samsung TV, discovering via Upnp and initilising the connection http://www.samsungdforum.com/Guide/tut00024/index.html#http-client-application

Latest messaging Guidelines http://www.samsungdforum.com/Guide/art00028/index.html
