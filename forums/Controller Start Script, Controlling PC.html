Andris,

are you running your controller on Linux or Windows? For Linux you will just have to link the "openremote.sh" script from tomcat/bin folder into /etc/init.d to have the controller start automatically. For windows it might be better if you use a standard tomcat installation which comes with an installer that creates the windows services which are needed to run java when starting windows. After that you just need to copy the controller.war file from tomcat/webapps folder into the new tomcat installation.

OpenRemote does not support running scripts on the controller invoked by commands. This is a good idea and we maybe you can create an issue in the OpenRemote jira (http://jira.openremote.org/).
