Hi.

I have a number of z-wave devices working now (soon I will NEED my pro license!!)

The latest addition to my network is a Horstmann room thermostat and central heating controller: http://www.horstmann.co.uk/central-heating.php#CentaurPlusZW

In general it is working with OR. I can change the threshold temperature on the thermostat and I can manually turn the boiler on and off using a switch however I do have a few issues that I guess are due to the implementation of z-wave in OR? All of these work fine in z-wave.me

The Thermostat returns both the threshold temperature AND the current room temperature however I can't seem to find a way to get the current temperature in OR? The temp command returns only the threshold temperature and status command always returns null (I think this is battery low though). In z-wave.me it provides both a thermostat and a sensor device (the sensor contains the current room temp).

Also the boiler controller is a switch. It doesn't seem to automatically return the current status when it is changed. This happens in z-wave.me as well but z-wave me has an "Update" button which does successfully get the current status of the switch.

While this is working well enough for me to "play" with, it means that any automation of the heating that I can do is limited as I have no way of telling the current temperature or whether the boiler is currently on or not?
