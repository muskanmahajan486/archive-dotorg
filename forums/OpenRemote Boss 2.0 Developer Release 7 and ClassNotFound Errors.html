This page last changed on Sep 27, 2010 by juha.

It seems several people are having issues with the 2.0 Developer Release 7 giving class not found errors.

I've reproduced at least one such case which was due to the scripts not handling whitespace in paths correctly. So if you're seeing this, please rename the directory "OpenRemote-Boss-2.0.0_DeveloperRelease 7" to "OpenRemote-Boss-2.0.0_DeveloperRelease7" and give it another try.

If you still are having the same issue after this, then post here on the forums and we'll do further debugging.
