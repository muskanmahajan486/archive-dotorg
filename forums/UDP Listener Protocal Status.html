Just timestamp for now. I will create a Howto for the UDPListener next and describe what the current implementation does.
The regex is only to match an incoming message and if the match is true the current timestamp is put into the sensor.
I did this to match on incoming messages from my IRtrans module to be able to control stuff using my apple ir remote.

The timestamp is used because the sensor would not trigger the rule engine if the new value equals the previous value.
My IRtrans always send the remote name and command name that was received and therefore only the first signal would execute a rule.
