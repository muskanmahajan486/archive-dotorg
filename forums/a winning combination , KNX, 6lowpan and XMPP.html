6LoWPAN on 2.4Ghz and 868MHz

6LoWPAN is an acronym of IPv6 over Low power Wireless Personal Area Networks. 6LoWPAN defines header compression mechanisms that allow IPv6 packets to be sent to and received from over 802.15.4 based networks. Thanks to this technology GenOHM end node devices could be accessed directly by IP address from any supported network, moreover you can configure them with DNS name,advanced security mechanisms, firewall, load-balancing and other TCP/IP networking features. You have much more control and more obvious work with 6LoWPAN networks rather than with ZigBee networks. It's also easier to integrate the network in monitoring and accounting applications.

Main advantages of 6LoWPAN

  • Full support of complex network topologies -  mesh, star, point-to-point, hybrid
  • Interoperability with all other IP-based networks
  • Possibility to name and route each separate sensor node to any IP-enabled device
  • Established TCP/IP network possibilities: addressing, naming, tools (Ping, Traceroute, SNMP), advanced security mechanisms (authentication, access control, data encryption, firewall) providing invulnerable data integrity, proxy architectures (Caching, Load balancing, NAT), application level data model and services (HTTP, HTML, XML, web services)

XMPP for communication with sensors

The Extensible Messaging and Presence Protocol (XMPP) is an open technology for real-time communication. In GenOHM modules XMPP is used to organize communication between GenOHM module and it's connected sensors. It allows for easy two-way communication, so no polling has to be used. This protocol is used since a huge count of sensors could be connected to one GenOHM device and for controlling them, standard serial tunnels will suffer from overload and inconvenient usage.
