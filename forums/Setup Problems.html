Well couple of things: Java 6 is required, looks like in both cases you are using Java 5 - whether or not that is affecting you is unsure but it's bound to be a glitch at some point.

On this release the runtime I've tested is Tomcat 6.0.18 – 6.0.20 sounds close enough that I wouldn't expect major issues but 5.5 seems like more questionable choice.

I'm regularly on Snow Leopard, Debian, WinXP so the OS should not be the issue.

Also, if you could try one more thing after the above updates, make sure you wipe out the work/ directory in your Tomcat installation (this causes issues for me often as Tomcat's redeployment behavior is ... a bit odd) and make sure your /webapps/controller/WEB-INF/web.xml file is still where it should be.
