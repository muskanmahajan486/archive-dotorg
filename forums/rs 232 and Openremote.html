Posted by phreak at Sep 30, 2013 20:22

how many Clients can OpenRemote have at once?

There's no hard limit. In practice it will depend on the hardware/operating system you choose to host the controller in, and how many concurrent threads that environment can support (and ultimately how much memory is made available, since each thread will allocate some memory for its call stack).

Posted by juha at Oct 03, 2013 17:40

The Marantz protocol is IP based and should also work with latest Denon AVRs.
But in the current implementation, the definition of the IP address of the AVR is global to the OR controller, so only 1 AVR can be controlled.
It could be possible to include an IP in the command (this for instance has been done for the Samsung protocol), but it's kind of a patch waiting for the "connectivity layer / gateway" concept that should solve these kind of issues globally.

The Denon protocol goes over serial but I believe would have the same limitation as, IIRC, the serial port is defined globally per controller.

Posted by phreak at Oct 04, 2013 17:19

The IR import on the Pro designer allows you to upload a Philips Pronto Pro XCF file to the server, and convert the non protected IR codes into commands suitable to control a Global Caché (TCP commands) or an IRTrans device (UDP commands).

Supported IR codes are learned raw formats + IIRC RC5, RC5x and Nec codes.
Codes coming from the Pronto database can not be used, as they are encrypted.
