Thank you for the support, i have solved the problem 3 days ago, changing the name of the war packet from Designer-2.13.0-2012-06-01-Rosemary.war to modeler.war. But there was actually another problem which i traced in the log file.
The problem was that UIComponent.java and BusinessEntity.java have different return types and the error i traced was:

"java name clash: list<BeanModel> createModels(Collection <? extends BusinessEntity> list) in BusinessEntity.java and list<BeanModel> createModels(Collection <? extends UIComponent> list)in UIComponent.java have the same erasure, yet neither hides the other"

I figured out that this concept is only allowed in java6 and java7 forbids it. After installing java6 instead of java7 i got problem solved.

Similiar question was posted on StackOverflow: http://stackoverflow.com/questions/6643648/java-static-method-binding-and-generics-all-rolled-up-with-some-method-overload
