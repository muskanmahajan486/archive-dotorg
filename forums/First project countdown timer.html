Zigbee is not yet supported in OR, so I describe a Zwave example. I assume here you install everything on a Windows PC.

For the experiment you could take a Zwave switch plugin (for example http://www.pepper1.net/zwavedb/device/53 )
Bear in mind that Openremote only has experimental Zwave support at the moment. So it is suggested you first setup your tiny system with the free cloud service http://zwave.me.
Once you have your controller (e.g. Aeon Zwave Stick S2) and the switch plugin working, stop the Zwave>me client software, and move to the openremote designer. (I assume you have read the instructions elsewhere on this forum)
Designer can detect and autoconfigure your Z-Wave Switch Plugin.
Once that is done create a macro with the name boiler. First select the "on" switch, next for a 10 minute duration insert a delay with the value of 600000, followed by dragging the "off" switch command.
Final step is in UI designer to make a screen with one button that refers to the macro boiler.
For smarter control you have to dive into the rules (Drools) which is perceived as more difficult.

Note: I have this working from my Android tablet. (I did get some strange behaviour in the beginning for which I have not yet an explanation)
