Hi,

I remember a chat about this. If I'm correct, a KNXnet/IP Router is a device which act as a gateway. It allows to forward telegrams from one media (usually TP1) to IP and vice versa.
But, as KNXNet/IP is non routeable (regarding multicast I guess), it was planned with the IP specification to have KNXNet routers, which function is to route these unrouteable KNXNet packets. So, it's a pure IP product, which would be reserved for large installations.
