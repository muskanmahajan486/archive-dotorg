Posted by pocster at Jan 24, 2013 11:19

For dynamic content, the only available UI element is a <web> widget in the control panel UIs.

Since programming is not an issue, that may be an option – pull your artist list into a html page and serve it over http URL which you point your <web> element to. If you need to control things from the web page, you can look at the controller's REST API on how to trigger actions through the controller.

Haven't tried it, so take it as a hobby project which may or may not work – it's a little short of building your own app from scratch but definitely requires some programming work.

I don't recall exactly if something for dynamic lists was already implemented on the web console, so you would have to try contacting Richard Turner for details.

Not familiar with any Python Z-Wave packages so can't comment there.

To clarify a little the points above, if your artist list is 'passive' in that you only want to display the content but not interact with it inside the remote control then AJAX and JSON are not relevant – you can leave interaction to already supported UI widgets in OpenRemote panels which use the API mentioned above for HTTP/REST requests.

If your dynamic web content however is intended to act on the data (artist list) somehow, then AJAX/JSON is relevant in order to create the necessary HTTP requests to the API mentioned above. You could look at the web console implementation for hints how to do this, as it uses the same API and generates HTTP requests from within a HTML/Javascript engine.
