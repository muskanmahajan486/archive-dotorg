If you've turned on bridged mode, the host and guest system should be in the same network (network part of the IP address is the same)
e.g:
host: 192.168.0.4
guest: 192.168.0.11

In order to test basic communication try to load the page http://192.168.0.11:8080/controller from a different machine in your network (e.g. your Android smartphone). If that's possible, there should be no problem for the Android app to load the panel list if you add the controller manually.

Posted by louise35 at Aug 28, 2013 10:36

I think you've tried to determine the IP of the guest with a terminal which is running in the host OS - I think that's not possible.

Determine IP of guest:

  1. Start guest operating system
  2. Log into guest operating system
  3. Start terminal and determine IP address

Do not care about the loopback interface with IP address 127.0.0.1 - it's a special address.

Btw: Are you really running the OpenRemote controller in a virtual machine (guest) on you Ubuntu (host) system or is there a misunderstanding ?
