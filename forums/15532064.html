Short answer:

Server is needed for automation. If you want your lights to turn on automatically at 7PM, you don't want to find a remote to press the button yourself to do so. That would only be remote control. Not automation. Or if you'd like to be alerted when the smoke detector goes off while you're away. These are all automation cases.

Remote controlling software like XBMC is a subset of this. It involves commands based on user interaction, not automation.

There's an effort underway to let you install a serverless version for the pure remote control use case.
