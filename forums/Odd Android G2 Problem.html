Interesting. If I remember correctly, I made a change (that's probably not available in the market yet) to allow the AppSettingsActivity (which is what you're using) to change orientation. I believe it was portrait-only before.

Dealing with orientation changes for that particular activity is a bit tricky, because it launches a background thread to auto-discover controllers, and an orientation change causes the currently running activity to be destroyed and then recreated. If the background thread still refers to the activity in its original incarnation and then the orientation changes, problems could ensue ...
