Posted by doucga at Nov 09, 2012 02:16

SOLVED!

I take it that Eric is away from the forums.

I finally found the solution and thought I'd share it with others that may run into this.

Eric's suggestion "In the OpenRemote SVN, under trunk/Runtime/apache-tomcat-6.0.18-SIP, you'll find a version of tomcat with Mobicents integrated. Using that container to deploy OR should fix that issue." was indeed the solution.

The iOS OpenRemote app is looking for a SIP connection on port 5060 when it starts up. If it can't register the connection then the dreaded "Registration Failure. Bad credentials, check your account settings" message pops up.

My problem was that I also have Asterisk running on the same server which also binds itself to port 5060 by default. In this case it is the same port the Mobicents servlet needs to listen on to be able to respond to the registration requests from iOS devices.

Once I put a different port number in FreePBX under Tools -> Asterisk SIP Settings -> Bind Port, Mobicents servlet was able to listen on port 5060 and my OR iOS apps no longer threw the error message. Hooray!
