Hi,
I have the following setup.
The lights in my bedroom turns on every morning at 6:30.
This is made possible by creating the following rule:

rule "WAKING UP BY TURNING ON LIGHT MASTER BEDROOM"

timer (cron: 0 30-45 6 ? * 2-6)

when eval(true)

Event( source == "Status - Lys - 2 etg - Master Bedroom - Morgenlys", value == "on" )

then

execute.command("Lys - Dimming - 2 etg - Master Bedroom (UP)", 1);

end

This works fine, but I am wondering, is it possible to make a App interaction with this rule or in some other way, so it is possible to change the timing of the wakeup.
For ex if I would like to sleep in late one day, I have to go into rules and change the time.
I would like to have some kind of interface on the OR App, that allows me to change the time.

Thanx

Comments:

If you wish to change timing of the wakeup then you must drop the timer() statement and add a wake-up sensor to the Event list. You can study my IoT challenge 2014 design when I set a bed time, just opposite to the wake-up time.
