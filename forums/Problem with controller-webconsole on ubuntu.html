Try with editing log4j.properties and config.properties to write logfiles and config to directories where you definitely have write-access. The defaults, the webapps folder, does not work in a standard Ubuntu tomcat6 setup.

Tomcat logfiles are at /var/lib/tomcat6/logs
Tomcat webapps dir is at /var/lib/tomcat6/webapps

Besides the log4j logs I found I often need to look at the catalina log.

You restart tomcat using the command sudo /etc/init.d/tomcat6 restart
