Are you using rules of any kind? I was under the impression that Drools required Java6 to run. If the hard-float Oracle JDK8 runs everything then that's probably big enough news to change the installation instructions on the OR main site.

EDIT: On a quick test over here, I replaced icedtea-6-jre with the Oracle hard-float preview. Seems rules are running, the server boot-up time was about 3x faster (20s vs. 60s) and panels are much more responsive. Warrants further testing, but initially this looks very, very good!

For everyone: http://jdk8.java.net/fxarmpreview/ <- download the raspberry hard-float JDK preview http://jdk8.java.net/fxarmpreview/javafx-arm-developer-preview.html <- installation instructions

http://www.openremote.org/display/docs/OpenRemote+2.0+How+To+-+Install+Controller+on+Raspberry+Pi <- running OR on Raspberry
The only change is the part where it says "export JAVA_HOME=/usr", but with the Oracle JDK it works with "export JAVA_HOME=/opt/jdk1.8.0/"

EDIT2: After a bit more of testing, the 20s vs. 60s difference was better explained by the lack of panels in a virgin OR installation. However, responsiveness of user input is observably snappier now than with icedtea.
