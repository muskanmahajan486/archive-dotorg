OpenRemote does need to fix the install instruction on the OR website as this problem drove me a little nuts at first. However, none of us are perfect so here is the way I figured out how to do it.

If you install OR from the OR website instructions there is an error in step 2 setting up the environment variable. According to OR you have to setup a Windows environment variable JAVA_HOME when it should be JRE_HOME. Here is a very basic install step-by-step.

I would install OR on a 32 bit OS but if you plan to only use it to access OSA then 64 but will work fine.
Download OR to your computer from http://www.openremote.org/display/docs/Get+Started
Download Java JDK 6 (NOT 7). OpenRemote does not explain very well that you must have the Java JDK not just the Java Runtime Environment. http://www.oracle.com/technetwork/java/javase/downloads/jdk6-jsp-136632.html

At this point you should have JDK and OR installed. On my machine JDK is install at "C:\Program Files\Java\jdk1.6.0_35" and OR is at "C:\OpenRemote-Controller-2.0.1".

Here is where OR has some errors. Before starting OR you have to create an environment variable with the variable name JRE_HOME and point it to the JDK install directory but do not include spaces. It should look something like this.

NAME: JRE_HOME
VALUE: C:\Progra~1\Java\jdk1.6.0_35

For 64 bit OS use the vaule C:\Progra~2\Java\jdk1.6.0_35

YOU MUST REBOOTE YOUR COMPUTER AT THIS POINT! That is also NOT in the OR instructions which caused me some time troubleshooting.

Now from the command prompt go to the OpenRemote bin directory "D:\OpenRemote-Controller-2.0.1\bin" and type "openremote start".

That's it.

Posted by mredeker at Jan 10, 2013 08:09

Here's what worked for me on Windows 8 32 bit:

  • Installed JDK to C:\Program Files\Java\jdk1.6.0_37
  • Added environment variable JAVA_HOME with value C:\Program Files\Java\jdk1.6.0_37
  • Appended C:\Program Files\Java\jdk1.6.0_37\bin to the Path environment variable setting

Note: after setting environment variables you have to open a new command prompt window to get the new environment variable setting - the environment variable setting of existing command windows is not refreshed --> reboot is not required
