Hi all,

For some project I've made recently (some of you may have seen it on Facebook or G+, for the other, here is a video on youtube), I've added SOAP as a Command/Sensor protocol within the OR controller.
The code is available in my svn workspace. The goal of this implementation was to use a third party gateway with OR. It was based on the latest trunk revision.

Here is a samble declaration in the controller.xml file :

<command id="568" protocol="soap">
<property name="wsdl" value="http://soapdomain.com/remoting/soap?wsdl" />
<property name="portName" value="/remoting/soapHttpPort" />
<property name="operationName" value="getValue" />
<property name="in0" value="temperatureSensor" />
<property name="timeout" value="300" />
</command>

The implementation is quite generic, using Axis toolkit. You should find it easy to use if familiar with SOAP. Thus, it was made for a particular context, and I'm not sure this may be useful for more realistic applications.

Mathieu
