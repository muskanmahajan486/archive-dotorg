Error code 36 comes from a KNX IP gateway if the connection is already in use. Most KNX IP gateways only allow one connection (a few newer models allow 4). If you are connected with ETS or another tool the OpenRemote controller cannot connect to KNX at the same time. Try and close ETS or anything else and start OpenRemote controller again.

Thank you Marcus

IFOUND THE PROBLEM I HAD: I had a "virtualbox Host-Only Network" networkcard with a fixed IP address and no gateway. Undoing this setting resolved the problem (selecting: obtain ip address automatically)

In the logs I could see the app trying to use the ip address of the virtual card to get to the KNX ip gateway that was in another range.
