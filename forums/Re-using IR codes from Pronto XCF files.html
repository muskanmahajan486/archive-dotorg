Hi all,

Trying to make working with IR codes in OpenRemote easier, we built an importer for Pronto XCF configuration files.
This will read in your XCF file and convert it to the appropriate commands to send IR through either a GlobalCaché (GC or iTach) or IRTrans device.
There are some formats that are not supported. Some we might support later based on the feedback received, some we won't be able to support (unless somebody can provide more insight into the format).

Here is a small video demoing this functionality: XCF IR Import.mov

For now, this is available in this branch: https://openremote.svn.sourceforge.net/svnroot/openremote/workspace/ebariaux/_Archive/UIDesigner_2_0_0_Alphas_IRImport

Hello Bengt,

The file that is imported in the video is an XCF file, such as those generated by the latest generation of ProntoEdit Professional for the TSU line of Pronto remotes.
The import will look at the IR codes available in the file and allow to create the appropriate commands for either GlobalCaché or IRTrans.
The codes can be in raw format, RC5, RC5x, Nec (I think I still need to add RC6 support).

The approach taken here it just to generate command for existing TCP or UDP protocols so that no change to the current controller is required.
A future direction will be to review the controller so that IR is "a first class citizen" with its own OR protocol in the controller.

XCF parsing and IR code format conversion is using an updated version of a library I did develop earlier and only the jars are included in svn, not the source code at this stage.
