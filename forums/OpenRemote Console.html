The iPhone app "OpenRemote Console" is the iPhone user interface for OpenRemote Boss. Besides the iPhone console we are developing an Android based console and a web console. Each of these consoles is working together with the server part of OpenRemote Boss which is the OpenRemote controller. The controller is a java based web-application running on Tomcat which communicates with your devices.

You need the Console on the iPhone and the Controller on a server to have a running system.

Posted by mredeker at May 27, 2010 20:14

If you just want to compile the app and run in iPhone Simulator, choose 'iPhone Simulator 3.x.x' as Active SDK and 'Debug' as Active Configuration. this needn't Code Sign.

Code Sign is used for running app in iPhone device, you should install your certificate file into KeyChain successfully on your Macbook, make sure you can find it in KeyChain Access, then create a provisioning file:

1.login dev center:http://developer.apple.com/iphone/index.action
2.choose a team
3.click 'iPhone developer program portal'
4.choose 'Provisioning' tab on the left
5.download & install(turn to 'how to' tab for further help)

a) download a active (not expired) provisioning profile
b) In XCode, Window->Organizer, connect iPhone/iPod touch to Macbook, select your device, in summary tab, add that provisioning profile;
c) In XCode, in the "Groups & Files" pane, expand "Targets" and double-click on your app's target. This will bring up the Info pane for your target. In the "Build" section, check the "code signing" section for any old profiles. replace 'iPhone Developer: Ryan Cai (xxxxxxxxxxx)' with your new one.

if you come across 'code sign error: provisioning profile 'XXXXX...' can't be found', turn to step 5.c

changing the project setting won't work, you have to change target setting.

if you are using Snow Leopard, please update iPhone console code from SVN to latest revision 2347. otherwise, a gcov error will break the build.
