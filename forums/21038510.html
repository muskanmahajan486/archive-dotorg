Hi Kevin,

There are scaling issues with the current Android app and this is due to display density difference between devices, Android (and iOS also) tries to be clever and it will scale the UI to try and maintain physical size across different display densities and as such it will trick the app and although you may have a display with 1024px x 768px if Android identifies it as having a display density of 2 then the app would think you have a 512px x 384px display.

We're looking at addressing this at the moment; on the iPhone for example with a retina display which has a display of 640px x 960px you still have to create a panel in the designer with a size of 320 x 480 because the display density is 2 (i.e. there are twice as many pixels per inch - how many pixels per inch is 'standard' is a decision made by the implementation - for web browsers it is 96dpi, for Android and iOS it is 160dpi - hence an iPhone screen is 2" x 3" in physical size).
