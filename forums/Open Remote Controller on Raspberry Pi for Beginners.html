My Raspberry Pi arrived a couple of days ago and I have been spending some time getting to know it. I took notes as I did the install, and I thought that I would share them and hopefully save someone else a few headaches.

This tutorial has been copied to the documentation section, where is more up to date OpenRemote 2.0 How To - Install Controller on Raspberry Pi

Install Rasbian

Go to http://www.raspberrypi.org/downloads and download the Raspbian "wheezy" zip file. Follow instructions on that website on how to install image onto SD card.

Install java 

sudo apt-get install icedtea-7-jre-cacao

Check java installation

$ which java
/usr/bin/java
$ java -version
OpenJDK Runtime Environment (IcedTea7 2.1.1) (7-u3-2.1.1-1+rpi1)
OpenJDK Zero VM (build 22.0-b10, mixed mode)

Set JAVA_HOME environmental variable

$ export JAVA_HOME=/usr

Install Controller

Download to Desktop Open Remote Controller 2.0 at openremote.org

Make openremote.sh executable. 

$ cd ~/Desktop/OpenRemote-Controller-2.0.0/bin
$ sudo chmod +x openremote.sh

Start the controller

$ ./openremote.sh run

Test Controller installation

Open browser and log onto http://localhost:8080/controller.

Start OR automatically at startup

Add a line to /etc/rc.local (East way)

Just before the exit 0 

/home/pi/Desktop/OpenRemote-Controller-2.0.0/bin/openremote.sh start

Or

Put a script file in /etc/init.d (Proper Way)

For example see http://www.openremote.org/pages/viewpage.action?pageId=20449784&focusedCommentId=20450152#comment-2045015

Make script executable

	
$ sudo chmod + /etc/init.d/NameOfYourScript

Make the script run at startup

$ sudo update-rc.d NameOfYourScript defaults

At this point I can open the Open Remote iphone app and the controller is automatically detected. I haven't really designed many panels yet, so I can't say how well the program works.

Hi everyone,

A couple of notes:

  • It seems slider widgets in panels don't show the current status properly with icedtea-7-jre-cacao, but with icedtea-6-jre-cacao they worked alright
  • Step 10 in the instructions leads to errors when you try to sync from online designer if everything is in one line. Instead split to two commands:

cd /home/pi/Desktop/OpenRemote-Controller-2.0.0/bin
./openremote.sh start

and this will fix the error (there appear to be some path assumptions in Openremote that fail if the process is not started in its own folder)
