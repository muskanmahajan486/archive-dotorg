Hi,

The Web Console 2.0 is a little misleading at the top of the toolbar (this should be changed); it referred to the fact that it was a complete replacement of the original web console that existed many years ago. The build number at the bottom of the toolbar is the important part and that is populated at build time based on the revision of the last commit within the branch being compiled.

Web console is versioned separately from the controller.

If you are seeing the same build number in different versions of the web console then this can only mean one of two things: -

-The versions (although labelled differently are actually the same)
-The ANT build task which gets the last commit revision was not working correctly when the WAR files were generated

I have checked the project tags for the web console and this is what should be: -

Web Console 2.1.0 - r7746
Web Console 2.1.2 - r8914
Web Console 2.1.2 (for Pro Controller 1.0.10) - r8961
Web Console 2.1.3 (I don't see a compiled WAR of this available in the downloads) - r9683
