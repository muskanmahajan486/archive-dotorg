Also, even when not present on the remote control bundle with a device, some devices have discrete on/off codes.
You can for install find them from Remote Central.

Also sometimes, devices will power up with another code than the power toggle, e.g. channel up. In that case, you can work you way around by using this and the power code.

The solution Johannes gave is indeed the "simple" way if the above does not work, but has the disadvantage that it can get out of sync if you control equipment with anything other than OpenRemote.

The solution to this is to use some form of sensor (light sensor on power led, power consumption, video sync, ...) to know to power state of a device. This becomes a bit more complex and costly.
