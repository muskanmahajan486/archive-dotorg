Posted by yannick at Jul 13, 2010 14:56

So a VPN is probably your best bet.  For outside access to services I usually forward ports instead of using a VPN, but I don't think the controller has any username/password setup so that could be a real security issue.  Do you have a VPN working already?  If not thats where you will need to start.

I use Linksys and Buffalo routers flashed with DD-WRT firmware.  This gives you a built in PPTP server which i believe is compatible with the iPhone vpn client, see http://www.dd-wrt.com/site/support/router-database to see if your router can be flashed with this firmware. 

You can also forward the correct ports back to a linux/mac/pc and run a vpn server there. 

Once you have a VPN, you will probably want to hard-code the controllers internal ip address on your phone as I don't know how well the muti-cast discovery will work on such a connection.

You will need to do some googling to figure out how to run a vpn server thats compatible with the iphone, I'm sure there's quite a few how-to's out there.

Yannick try this,

  1. Set up a free account with dyndns.org (I'm assuming you do not have a static ip)
  2. Configure your firewall/router with dyndns, or if its not compatible, download thier free software to your PC/MAC
  3. Forward port 3671 (usally under gaming and applications in most routers) to the ip address of the KNX hub
  4. use your dynsdns address to connect.
