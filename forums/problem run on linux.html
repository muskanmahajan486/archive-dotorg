Posted by juha at Feb 15, 2011 10:04

Also many of the OSS JavaVM's have issues with handling JAR manifest 'Classpath' entries properly which often shows up as java.lang.ClassNotFoundException: org/apache/juli/ClassLoaderLogManager, since this is how Tomcat bootstraps its startup classes (and logging).

Further I found those based on GNU Classpath do not have a complete Java 1.5 class library support which often prevents Tomcat from starting – related to lack of complete JMX implementation IIRC, there was an open issue wrt to that on GNU classpath issue tracker but last I checked (a year or so ago) which was never addressed.

GNU Classpath seems to have stalled somewhat since OpenJDK started. I would recommend using a JVM compiled against OpenJDK class libraries instead of GNU Classpath (you will still need Java 1.6 compatibility too though).

In OSS, ZeroVM seems to be most up-to-date related to embedded and non-x86 based CPU support (http://openjdk.java.net/projects/zero/). It still needs a proper JIT though for things to run at good speeds (project Shark).

HTH,

– Juha
