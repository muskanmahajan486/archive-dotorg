Ivan,

You will need to proxy the HTTP requests to your controller (the consoles use all HTTP protocol). You can do this by turning off the auto-discovery in the iPhone client and entering a publicly available domain URL (ie. with DNS entry) in the phone manually.

For example: controller.mydomain.com/controller

Then proxy the HTTP requests coming to this URL to the controller instance.

There's probably more than one way to skin this cat (iptables, squid, apache httpd): choose whatever you're most familiar with, I personally do it with apache httpd + mod_jk (a heavy approach for mere http forwarding but since I have apache and mod_jk already set up for other purposes, easy to adapt to this one too).

So the flow is: 

iPhone <--- HTTP ---> httpd or proxy or port forwarding at public IP <--- HTTP ---> OR Controller

The details will depend on what you prefer to use to forward the HTTP requests. But basically this is no different than setting up any other web application server that is behind a firewall.
