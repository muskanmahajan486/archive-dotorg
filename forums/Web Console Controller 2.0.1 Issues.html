Some users have reported issues with the Web Console getting stuck at the Loading screen when trying to connect to their controllers.

The only possible cause of this I am aware of is if you are using security on your controller and you are using Controller version 2.0.1 or older.

Due to issues with the Controller REST API the Web Console won't be able to connect to Controller versions 2.0.1 or older; I created a few patches to fix the problem which will probably be included in Controller 2.1.0.

In the meantime I have a Controller branch in my workspace that has the patches applied which you can use but this is based on Controller 2.0.0: -

https://openremote.svn.sourceforge.net/svnroot/openremote/workspace/richturner/Controller_2_0_0_patched

I've also compiled this branch and put it on my server so you can download the zip from there if you'd rather: -

http://multimation.co.uk:8080/controller/OpenRemote-Controller-2.0.0_Patched.zip

This should allow you to use the Web Console with a secured Controller.
