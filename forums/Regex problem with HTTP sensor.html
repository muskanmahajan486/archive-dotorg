Hello,

I'm a big enthusiast of OR. Now I'm trying to integrate it with my irrigation system, which can be controlled through the HTTP protocol.

I'm having problems in order to get the status of the Irrigation controller out of the HTML returned, using a regular expression.

HTML returned:

<html>
<head>
</head>
<body>
EtherRain Device Status <br>
un:EtherRain 8

ac: <br>
os: RD <br>
cs: OK <br>
rz: UK <br>
ri: 0 <br>
rn: 0 <br>
</body>
</html>

Regex: os: (..)

I'm not expert, at all, with regex, but when I try to pull out just the two final letters of the match by grouping them with (), I always get the whole match, including the first part 'os: '.

Is it possible to get only the last part ('RD' in the example above)?

This should work:
(?<=os\:\s)(\w*)

Also if output can be only 2 char then use

Unknown macro: {2}
instead of *
Posted by reznetwork at Jun 14, 2013 11:18

Thank you very much!!!

Yes, Alex, you're right. It works fine!! As I'm not used to regular expressions, I didn't know that you can avoid capturing a group by using the special construction (?<=X), where X is the group you don't want to capture.

That's the solution as the OR Designer does not allow you to select the matched group of the regex in the UI
