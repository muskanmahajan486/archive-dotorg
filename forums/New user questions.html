Hi Thierry,

The Android panel screen size is a bit tricky at the moment – since we are using API 3 the different screen sizes must be specified explicitly in the configuration which is not working too well with custom panel sizes.

This will be a trivial fix when upgrading to a later Android API's (in fact Eric Bariaux already fixed this I believe) although I still need to look into the details.

So at the moment, the current Android client version is more or less a fixed resolution. I'll look at it once I get back to making changes to the Android client.
