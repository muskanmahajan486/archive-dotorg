Keyword Search

The original Beehive has a 'search' (more like navigation) that exposes data as you first choose a vendor and then a model type. So similar to navigating a filesystem or other tree structure.

We should have a direct key phrase search. This can be used to search model numbers directly but also used to search the various infrared flag fields stored in the database schema.

The search functionality should be made available on the Beehive web UI but also as a REST API for other applications to integrate with and search Beehive contents. Java API for search should be provided for programmatic access.

Deliverables

  1. Java API for executing a keyword search against the Beehive SQL database
  2. REST API for executing a keyword search against the Beehive SQL database
  3. Ability to enter a keyword search in the Beehive web user interface
