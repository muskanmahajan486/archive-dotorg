Project Information : What is included in OpenRemote 1.0 Milestone 2?
This page last changed on Jun 16, 2009 by juha.

OpenRemote 1.0 Milestone 2 is a developer release which includes an end-to-end infrared implementation. All the components to run an user interface on an iPhone or iPod Touch and a controller to translate HTTP requests to infrared (LIRC) codes are included.

The included functionality is demonstrated on the Infrared Demo Video and It works! blog. Milestone 2 extends the functionality shown in these videos with a more customizable user interface. You can see examples of types of user interfaces that can be created with M2 in this blog entry.

Milestone 2 has been deployed and is available online at http://composer.openremote.org/1.0/M2 for demo purposes.

New features include:

  • Ability to create multiple screens in iPhone user interface layout
  • Grid size in the iPhone UI layout can be altered
  • Button size can be altered to span multiple rows or columns in the grid
  • Button labels can be altered from their defaults or translated to other languages
  • Button labels can be replaced by icon images
  • Default icon theme can be configured for the UI Composer
  • Custom user icons and images can be uploaded to the iPhone console layout
  • Macros can be nested

For more details on the above features, see What is new compared to OpenRemote 1.0 Milestone 1\?.

To deploy OpenRemote 1.0 Milestone 2 you will need a Linux-based computer with Java VM, Tomcat and LIRC installed. We've tested M2 with CommandIR II infrared transmitter. Other LIRC compatible IR transmitters may also work but we can't make any guarantees at this point. The hardware/operating system/infrared combination we've tested M2 with is ALIX/Voyage/CommandIR (see OpenRemote RI).

What is not included

OpenRemote 1.0 Milestone 2 does not include any other protocol functionality. While the UI composer does show tabs for generating events to other protocols such as KNX, X10, Telnet and HTTP, these features have not yet been merged to the milestone branch. In the meanwhile developers can review these features in KNXControllerEvent and TCP-HTTP-TelnetEvents feature branches. Both these branches have been updated to M2 revisions, and will be included in the next M3 release.
