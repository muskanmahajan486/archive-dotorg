Project Information : What is included in OpenRemote 1.0 Milestone 3?
This page last changed on Aug 19, 2009 by juha.

OpenRemote Boss 1.0 Milestone 3 is a developer release which includes an end-to-end infrared implementation and support for KNX and certain IP-based protocols. Apple iPhone and iPod Touch are supported as panel hardware. The recently announced Android panel implementation should still be considered alpha-quality and is not included in this release (but is available separately from subversion repository).

Milestone 3 is the last milestone before OR Boss 1.0 release. It is largely feature-complete with an exception of X10 library integration and some tweaks to KNX functionality. Fully supported protocols include infrared via LIRC (requires Linux as the OS platform), KNX, TCP/IP, HTTP and Telnet for IP-enabled devices. These protocols are supported as unidirectional messages to trigger commands. Bidirectional protocols will not yet be supported in the Boss 1.0 release.

OpenRemote Boss 1.0 M3 has been deployed and the User Interface Composer is available online at http://composer.openremote.org/demo. A stand-alone controller binary can be downloaded here. The stand-alone controller binary is a pure Java version and as such requires that LIRC to be installed separately.

New features in M3 include:

  • Controller auto-discovery from iPhone or iPod Touch consoles.
  • KNX, TCP/IP, HTTP and Telnet protocol support in UI Composer.
  • Macro delay configuration

Features added in previous milestone:

  • Ability to create multiple screens in iPhone user interface layout
  • Grid size in the iPhone UI layout can be altered
  • Button size can be altered to span multiple rows or columns in the grid
  • Button labels can be altered from their defaults or translated to other languages
  • Button labels can be replaced by icon images
  • Default icon theme can be configured for the UI Composer
  • Custom user icons and images can be uploaded to the iPhone console layout
  • Macros can be nested

What is not included

UPnP protocol support (contributed by Mathieu Gallissot) and GC protocol code (from Marc Fleury and Eric Bariaux) is not included in this milestone. You should expect to see them in 1.1 and/or 2.0 release of OR Boss.

All protocol implementations are unidirectional. Bidirectional protocols (e.g. status updates to iPhone console) are not yet supported.

Android console application has not been completely tested against OR Boss 1.0 M3. While the Android implementation is expected to work against M3 it should still be considered untested and experimental.
