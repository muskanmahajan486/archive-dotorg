Due to the use of CSS Transforms in particular rotations there seems to be discrepancies between how the various browsers return the position of elements that have had a transform applied to them.

WebKit based browsers (i.e. iOS Safari, Android, Google Chrome) - Return the position of the element in the transformed position.

Other Browsers - Return the position of the element in the position without transform applied.

For our purpose the WebKit method is the preferred behaviour and at present Web Console 2.0 only caters for these browsers.

There is currently an open bug on GWT here for some sort of consistency within GWT on element positions but as yet it is unclear as to what the outcome will be.

It is possible to examine the User Agent string and use different methods for each and this needs to be considered at some point.
